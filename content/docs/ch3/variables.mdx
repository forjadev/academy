Quando começamos a programar, é fundamental entender como armazenar e manipular informações, pois elas são o núcleo de qualquer aplicação. É aqui que entram as variáveis e os tipos de dados. Nesta seção, vamos conhecer o que são variáveis, como elas funcionam no JavaScript, e os diferentes tipos de dados que podemos usar.

## O que são Variáveis?

Uma variável é um espaço de memória onde armazenamos dados temporários para usarmos ao longo de um programa. Podemos pensar em uma variável como uma "caixa" que guarda uma informação, como um número, uma palavra, ou mesmo uma estrutura mais complexa. Variáveis nos permitem reutilizar dados e usá-los em diferentes partes do código.

Por exemplo, imagine que você quer calcular o preço total de um produto em um site. Você pode armazenar o preço de cada item em uma variável, realizar os cálculos e exibir o resultado. Em JavaScript, as variáveis são definidas por palavras-chave que indicam a maneira como o valor será manipulado e mantido na memória.

## Declaração de Variáveis

Para declarar uma variável em JavaScript, usamos as palavras-chave `var`, `let`, ou `const`:

- **`let`**: permite criar uma variável cujo valor pode ser alterado ao longo do programa. Esta é uma escolha segura e flexível para a maioria das variáveis.
- **`const`**: define uma variável cujo valor não pode ser alterado após sua atribuição inicial, ideal para dados constantes.
- **`var`**: uma forma mais antiga e menos recomendada de declarar variáveis. Devido a algumas particularidades de escopo, `var` pode ser menos previsível e, por isso, `let` e `const` são preferidos em códigos modernos.

Exemplo de declaração:

```jsx
let nome = "Ana";       // usando let
const idade = 30;       // usando const
var cidade = "Lisboa";  // usando var
```

## Regras de Nomeação de Variáveis

JavaScript tem algumas regras e boas práticas para definir nomes de variáveis:

1. **Deve começar com uma letra, underscore (_), ou cifrão ($)**. Não pode começar com números.
2. **Não pode conter espaços ou caracteres especiais (exceto _ e $)**.
3. **Evite palavras reservadas**, como `function`, `return`, ou `if`.
4. **Use nomes descritivos** para facilitar a leitura do código. É recomendável usar o estilo camelCase, como `precoProduto`.

Por exemplo:

```jsx
let precoProduto = 59.99;
let _nomeUsuario = "João";
let $valorTotal = 150.50;
```

## Tipos de Dados em JavaScript

Em JavaScript, uma variável pode armazenar diferentes tipos de dados. Isso é importante porque cada tipo de dado ocupa uma parte da memória e possui comportamentos específicos. Vamos ver os principais tipos de dados em JavaScript.

### 1. Number

O tipo `number` representa tanto números inteiros como decimais. Por exemplo:

```jsx
let idade = 25;       // número inteiro
let altura = 1.75;    // número decimal
```

### 2. String

Strings são sequências de caracteres usadas para representar texto. Elas são definidas entre aspas simples `'...'`, aspas duplas `"..."`, ou crases ``...`` (template strings). Strings podem conter letras, números, e caracteres especiais:

```jsx
let nome = "Alice";
let saudacao = `Olá, ${nome}!`;  // usando template string para interpolação
```

### 3. Boolean

O tipo `boolean` representa valores de verdadeiro (`true`) ou falso (`false`). É utilizado para expressar condições, como a validação de um login ou a verificação de uma idade mínima.

```jsx
let maiorDeIdade = true;
let cadastroCompleto = false;
```

### 4. Null e Undefined

- **`null`**: representa uma ausência intencional de valor. É usado quando queremos indicar que uma variável não tem valor.
- **`undefined`**: indica que uma variável foi declarada, mas ainda não possui um valor atribuído.

```jsx
let valorNulo = null;
let valorNaoDefinido;  // automaticamente definido como undefined
```

---

*Existem outros tipos em Javascript, mas por enquanto focaremos nesses tipos básicos. Eles são suficientes para entender os conceitos fundamentais de programação e começar a criar aplicações simples.*

---

Além disso, é importante notar que JavaScript é uma linguagem de tipagem dinâmica, o que significa que uma variável pode mudar de tipo durante a execução do programa. Por exemplo:

```jsx
let valor = 42;        // number
valor = "quarenta e dois";  // agora é uma string
valor = true;          // agora é um boolean
```

Essa flexibilidade pode ser útil, mas também requer cuidado para evitar erros inesperados no código.

## Operações com Variáveis e Tipos de Dados

No JavaScript, as variáveis não apenas armazenam dados, mas também podem ser usadas para realizar várias operações. A forma como essas operações funcionam depende do tipo de dado que a variável contém. Nesta seção, vamos explorar as operações mais comuns que você pode realizar com variáveis e como elas se comportam com diferentes tipos de dados.

### 1. Operações Aritméticas

Operações aritméticas são usadas para fazer cálculos com números. No JavaScript, você pode realizar operações como soma, subtração, multiplicação, divisão, entre outras.

**Operadores Aritméticos Comuns**:

- **Soma (`+`)**: adiciona dois valores numéricos. Se pelo menos um dos valores for uma string, a operação de soma se torna uma **concatenação de strings**.
    
    ```jsx
    let a = 10;
    let b = 5;
    let soma = a + b;  // soma é 15
    
    let texto = "A soma é: ";
    let resultadoSoma = texto + (a + b);  // concatena a string com o resultado da soma, gerando "A soma é: 15"
    ```
    
- **Subtração (``)**: subtrai um número do outro. Esse operador só faz sentido quando os valores envolvidos são numéricos.
    
    ```jsx
    let subtracao = a - b;  // subtrai 5 de 10, resultando em 5
    ```
    
- **Multiplicação (``)**: multiplica dois números.
    
    ```jsx
    let multiplicacao = a * b;  // multiplica 10 por 5, resultando em 50
    ```
    
- **Divisão (`/`)**: divide um número por outro. Caso a divisão resulte em um número decimal, o JavaScript o exibirá como tal.
    
    ```jsx
    let divisao = a / b;  // divide 10 por 5, resultando em 2
    let resultadoDecimal = 7 / 3;  // resultado será 2.3333...
    ```
    
- **Módulo (`%`)**: retorna o **resto da divisão** entre dois números.
    
    ```jsx
    let modulo = a % b;  // 10 dividido por 5 dá resto 0
    let outroModulo = 7 % 3;  // 7 dividido por 3 dá resto 1
    ```
    
- **Exponenciação (`*`)**: eleva um número à potência de outro.
    
    ```jsx
    let potencia = 2 ** 3;  // 2 elevado à potência de 3 resulta em 8
    ```
    

### 2. Concatenação de Strings

Em JavaScript, a operação de soma (`+`) tem um comportamento especial quando usada com strings. Se um dos operandos for uma string, o JavaScript não fará uma operação aritmética, mas sim uma **concatenação de strings**.

**Exemplo**:

```jsx
let nome = "Maria";
let saudacao = "Olá, " + nome + "!";  // concatena a string "Olá, " com o valor de nome e "!" no final
console.log(saudacao);  // "Olá, Maria!"
```

Além disso, você pode usar **template literals** (template strings) para realizar uma concatenação de forma mais legível. Com os templates, você pode embutir variáveis dentro de strings usando `${}`.

**Exemplo com template literals**:

```jsx
let nome = "Maria";
let saudacao = `Olá, ${nome}!`;  // mais fácil de ler e manter
console.log(saudacao);  // "Olá, Maria!"
```

### 3. Operadores de Comparação

Operadores de comparação são usados para comparar valores. Eles retornam um valor booleano (`true` ou `false`), que indica se a comparação é verdadeira ou falsa.

**Operadores de comparação mais comuns**:

- **Igualdade (==)**: verifica se dois valores são iguais, mas **não considera o tipo**.
    
    ```jsx
    console.log(5 == '5');  // true, porque o valor é o mesmo, mas os tipos podem ser diferentes
    ```
    
- **Igualdade estrita (===)**: verifica se dois valores são **iguais e do mesmo tipo**.
    
    ```jsx
    console.log(5 === '5');  // false, porque o tipo é diferente (número vs string)
    ```
    
- **Desigualdade (!=)**: verifica se dois valores **não são iguais**, ignorando o tipo.
    
    ```jsx
    console.log(5 != '5');  // false, porque o valor é o mesmo, apesar dos tipos diferentes
    ```
    
- **Desigualdade estrita (!==)**: verifica se dois valores são **diferentes ou de tipos diferentes**.
    
    ```jsx
    console.log(5 !== '5');  // true, porque um é número e o outro é string
    ```
    
- **Maior que (>)** e **Menor que (\<)**: compara valores numéricos e retorna verdadeiro ou falso.
    
    ```jsx
    console.log(10 > 5);  // true
    console.log(5 < 3);   // false
    ```
    
- **Maior ou igual (>=)** e **Menor ou igual (\<=)**: verifica se o valor é maior ou igual, ou menor ou igual.
    
    ```jsx
    console.log(10 >= 10);  // true
    console.log(3 <= 2);    // false
    ```
    

### 4. Operadores Unários

Os operadores unários operam em apenas um operando. O exemplo mais comum é o operador de incremento (`++`) e o operador de decremento (`--`), que aumentam ou diminuem o valor de uma variável em 1, respectivamente.

- **Incremento (`++`)**: adiciona 1 ao valor de uma variável.
    
    ```jsx
    let contador = 10;
    contador++;  // agora contador é 11
    ```
    
- **Decremento (`-`)**: subtrai 1 do valor de uma variável.
    
    ```jsx
    let contador = 10;
    contador--;  // agora contador é 9
    ```
    

Esses operadores podem ser usados tanto antes (prefixo) quanto depois (posfixo) da variável, o que altera ligeiramente o comportamento:

```jsx
let x = 5;
console.log(++x);  // 6, o incremento é feito antes da impressão
console.log(x++);  // 6, o valor é impresso antes de ser incrementado, então x se torna 7 depois
```

## Conversão de Tipos

No JavaScript, os tipos de dados podem ser convertidos automaticamente ou explicitamente, dependendo do contexto em que os valores são usados. As conversões de tipos são um conceito fundamental, pois o JavaScript é uma linguagem de tipagem fraca, o que significa que ele tenta fazer essas conversões de forma automática sempre que possível, o que pode levar a comportamentos inesperados, mas também facilita a escrita de código em muitos casos.

### 1. Conversões Implícitas (ou Automáticas)

As conversões implícitas acontecem automaticamente quando o JavaScript tenta "ajustar" os tipos de dados para realizar operações. Isso ocorre em várias situações, como quando você tenta fazer uma operação entre valores de tipos diferentes, e o JavaScript os converte para um tipo compatível.

**Exemplo 1: Soma de String e Número**

O operador de soma (`+`) é um dos exemplos mais comuns de conversão implícita. Quando um número é somado a uma string, o JavaScript converte o número para uma string, pois o operador de soma pode ter o objetivo de concatenar textos.

```jsx
let numero = 10;
let texto = "O valor é: ";
let resultado = texto + numero;  // A variável 'numero' será convertida para string
console.log(resultado);  // "O valor é: 10"
```

**Exemplo 2: Concatenação de Booleano com String**

Quando um valor booleano é concatenado com uma string, o JavaScript converte automaticamente o valor booleano em uma string correspondente ("true" ou "false").

```jsx
let status = true;
let mensagem = "O status é: " + status;  // 'true' será convertido para string
console.log(mensagem);  // "O status é: true"
```

**Exemplo 3: Conversão de Tipos em Comparações**

Em comparações, o JavaScript também pode fazer conversões implícitas de tipos, especialmente com o operador `==` (igualdade). Se os tipos forem diferentes, o JavaScript tenta converter os valores para que ambos os lados da comparação sejam do mesmo tipo.

```jsx
console.log(5 == '5');  // true, porque o JavaScript converte a string '5' para o número 5
```

No exemplo acima, mesmo que um dos valores seja uma string e o outro seja um número, o operador `==` faz a conversão implícita e compara os valores depois da conversão.

### 2. Conversões Explícitas (ou Manuais)

Embora o JavaScript faça muitas conversões de tipo automaticamente, em algumas situações é necessário que o programador faça a conversão de tipos de forma explícita. Para isso, existem várias funções que permitem transformar valores de um tipo para outro de maneira controlada.

**Funções de Conversão Explícita:**

- **`Number()`**: converte um valor para número. Se o valor não puder ser convertido para um número válido, o resultado será `NaN` (Not a Number).
    
    ```jsx
    let valor = "123";
    let numero = Number(valor);  // converte a string "123" para o número 123
    console.log(numero);  // 123
    ```
    
    Caso o valor não seja uma string numérica válida, o resultado será `NaN`.
    
    ```jsx
    let invalido = "abc";
    let numeroInvalido = Number(invalido);  // tentativa de converter algo que não é um número
    console.log(numeroInvalido);  // NaN
    ```
    
- **`String()`**: converte qualquer tipo de valor para uma string. Isso pode ser útil quando você deseja garantir que o valor seja tratado como texto.
    
    ```jsx
    let numero = 100;
    let texto = String(numero);  // converte o número 100 para a string "100"
    console.log(texto);  // "100"
    ```
    
- **`Boolean()`**: converte um valor para booleano (`true` ou `false`). No JavaScript, qualquer valor que não seja `undefined`, `null`, `0`, `NaN`, uma string vazia `""` ou `false` será convertido para `true`. Esses valores são chamados de "valores truthy". Os outros valores são chamados de "valores falsy", e eles se tornam `false` quando convertidos.
    
    ```jsx
    let numero1 = 5;
    let booleano1 = Boolean(numero1);  // qualquer número diferente de 0 se torna true
    console.log(booleano1);  // true
    
    let numero2 = 0;
    let booleano2 = Boolean(numero2);  // 0 se torna false
    console.log(booleano2);  // false
    ```
    
- **`parseInt()`**: converte uma string em um número inteiro. Ele tenta converter a parte da string que representa um número até encontrar um caractere que não faz parte do número.
    
    ```jsx
    let textoNumerico = "123abc";
    let numeroInteiro = parseInt(textoNumerico);  // converte a parte numérica da string
    console.log(numeroInteiro);  // 123
    ```
    
    Se a string não começar com um número, o `parseInt()` retorna `NaN`.
    
    ```jsx
    let textoNaoNumerico = "abc123";
    let numeroInvalido = parseInt(textoNaoNumerico);  // não consegue encontrar um número no início
    consol.log(numeroInvalido);  // NaN
    ```
    
- **`parseFloat()`**: converte uma string em um número de ponto flutuante (decimal). Ele funciona de forma similar ao `parseInt()`, mas com maior precisão para números decimais.
    
    ```jsx
    let numeroDecimal = parseFloat("10.5");
    console.log(numeroDecimal);  // 10.5
    ```
    

### 3. Conversões Implicitas e Comparação com `==` e `===`

Em JavaScript, é importante lembrar que a comparação de valores pode ser feita de duas formas principais:

- **`==` (igualdade não estrita)**: compara os valores, mas faz a conversão de tipo implícita se necessário.
    
    ```jsx
    console.log(5 == '5');  // true, porque o JavaScript converte a string '5' para o número 5 antes de comparar
    ```
    
- **`===` (igualdade estrita)**: compara tanto os valores quanto os tipos, sem realizar conversões implícitas. Ou seja, os dois valores precisam ser do mesmo tipo e ter o mesmo valor para serem considerados iguais.
    
    ```jsx
    console.log(5 === '5');  // false, porque um é um número e o outro é uma string
    ```
    

### 4. Conversões em Operações Matemáticas

Durante operações matemáticas, o JavaScript pode tentar converter valores automaticamente para garantir que a operação tenha sentido. Se um valor não for numérico, o JavaScript pode tentar convertê-lo para um número para poder realizar a operação.

**Exemplo 1: Operação com Strings e Números**

```jsx
let a = "10";
let b = 5;
let resultado = a * b;  // "10" é convertido para número e a operação de multiplicação é realizada
console.log(resultado);  // 50
```

**Exemplo 2: Adição de String e Número**

Ao tentar somar uma string com um número, o JavaScript irá converter o número para uma string para realizar a concatenação:

```jsx
let a = "10";
let b = 5;
let resultado = a + b;  // "10" é tratado como uma string e 5 será convertido para string
console.log(resultado);  // "105" (não é 15 porque o operador + realiza a concatenação de strings)
```
